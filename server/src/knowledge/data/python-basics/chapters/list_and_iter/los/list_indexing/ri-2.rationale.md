正确答案是 **`[20, 30]`**。

切片 `a[i:j]` 是**左闭右开**:从 index `i` 开始,**到 `j` 之前停止**,**不包括 `j`**。

- `a[1:3]` 取 index 1 和 2,共 **2 个**元素
- 数量 = `j - i = 3 - 1 = 2`,不需要"是否含端点"的纠结记忆

为什么这么设计?有两个实用理由:

1. **切片长度计算简单**:`a[i:j]` 的长度恰好是 `j - i`(假设没越界)
2. **拼接不重叠**:`a[:k] + a[k:]` 一定等于 `a`,中间没漏没重

这跟 C/C++ 的 `for (i = start; i < end; i++)` 是同一种约定 — 比"含端点"的写法更难写出 off-by-one bug。

从其他语言迁移的人最容易在这里翻车。多写几次 `a[1:3]` → `[20, 30]` 把肌肉记忆建起来。
