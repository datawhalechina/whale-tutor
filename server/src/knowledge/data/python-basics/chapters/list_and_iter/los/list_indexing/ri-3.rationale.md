正确答案是 **B:`a[5]` 抛 `IndexError`,`a[5:10]` 返回 `[]`**。

两种越界行为不同:

- **索引** `a[5]`: 在长度为 3 的 list 上抛 **`IndexError: list index out of range`**。这是显式失败,通常意味着代码逻辑出错,要崩出来让你修
- **切片** `a[5:10]`: **静默返回空 list `[]`**。设计哲学是"超出部分自动截断"。

为什么切片不报错?因为切片常用于"取一段尽可能多的元素",不知道列表实际长度时也能用,例如:

```python
def first_n(items, n):
    return items[:n]   # 即使 n > len(items) 也安全,返回整个 items
```

如果切片越界报错,这种用法就要每次手写边界检查,代码会很冗长。

记法:**单点要精准(越界即崩),区段允许容错(越界返空)**。

注意 `a[5:10]` 在 list 长度为 0 时同样返回 `[]` — 这是"切片永远返回 list,即使空"的另一面。
